# REGRAS
Regras e boas práticas para usar nossa organização da melhor forma


## Desafios

### Repositórios templates

Antes de criar um novo repositório com desafio para algum cadidato, verifique se você pode reaproveitar os desafios já presentes em outros repositórios. Temos vários desafios para fullstack, react, frontend, data engineer, etc. Só crie um repositório novo se for realmente necessário.

Repositórios de desafios devem ser **privados** e **templates**

**OBS:** Evite colocar número de vagas no nome do repositório ou na descrição do desafio. Por exemplo: `Desafio para  a vaga #GDP-XXX...` ou `gdp-xxx-challenge`. Isso permite que esse mesmo template seja utilizado em outras vagas semelhantes.

### Repositórios para candidatos

Para aplicar um desafio para um candidato, siga o passos abaixo:

  1. Escolha um repositório template e crie um novo repositório privado para cada um dos seus candidatos a partir desse template
  2. Adicione somente o candidato como colaborador nesse repositório recem criado
  3. Informe que o candidato deverá submeter um Pull Request com todo o código relacionado ao desafio
  4. Faça a review do código do canditado, forneça feedbacks!
  5. Quando a vaga for preenchida, delete ou arquive o repositório. 

Repositórios esquecidos abertos com mais de 2 meses serão deletados. :)
